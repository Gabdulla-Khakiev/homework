## Использование
### 1. Маскирование данных (masks.py)
#### Маскирование номера карты
Функция `get_mask_card_number(card_number: str) -> str` возвращает замаскированный номер карты, где символы с 7 по 12 заменяются на звёздочки (`*`).

Пример использования:
```python
from masks import get_mask_card_number

card_number = "1234567812345678"
masked_card = get_mask_card_number(card_number)
print(masked_card)  # Вывод: 1234 56** **** 5678
```
#### Маскирование номера счета
Функция `get_mask_account(account_number: str) -> str` возвращает замаскированный номер счета, где первые два символа последних шести цифр заменяются на звёздочки (`*`).

Пример использования:
```python
from masks import get_mask_account

account_number = "9876543210"
masked_account = get_mask_account(account_number)
print(masked_account)  # Вывод: **76543210

```
### 2. Обработка данных (processing.py)
#### Фильтрация по состоянию
Функция `filter_by_state(list_of_dict: List[dict], state='EXECUTED') -> List[dict]` возвращает список словарей, отфильтрованных по значению ключа `state`.

Пример использования:
```python
from processing import filter_by_state

data = [{'state': 'EXECUTED', 'date': '2023-01-01'}, {'state': 'PENDING', 'date': '2023-02-01'}]
filtered_data = filter_by_state(data)
print(filtered_data)  # Вывод: [{'state': 'EXECUTED', 'date': '2023-01-01'}]
```
#### Сортировка по дате
Функция `sort_by_date(list_of_dict: List[dict], reverse=True)` сортирует список словарей по значению ключа `date` в порядке убывания (по умолчанию).

Пример использования:
```python
from processing import sort_by_date

data = [{'state': 'EXECUTED', 'date': '2023-01-01'}, {'state': 'PENDING', 'date': '2023-02-01'}]
sorted_data = sort_by_date(data)
print(sorted_data)  # Вывод: [{'state': 'PENDING', 'date': '2023-02-01'}, {'state': 'EXECUTED', 'date': '2023-01-01'}]
```

### 3. Маскирование и форматирование данных (widget.py)
#### Маскирование данных в строке
Функция `mask_account_card(user_information: str) -> str` маскирует номер карты или счета в переданной строке.

Пример использования:
```python
from widget import mask_account_card

info = "Card number 1234567812345678"
masked_info = mask_account_card(info)
print(masked_info)  # Вывод: Card number 1234 56** **** 5678
```
#### Преобразование формата даты
Функция `get_date(date: str) -> str` преобразует дату из формата `YYYY-MM-DD` в формат `DD.MM.YYYY`.

Пример использования:
```python
from widget import get_date

date_str = "2023-08-10"
formatted_date = get_date(date_str)
print(formatted_date)  # Вывод: 10.08.2023
```

## Декораторы (decorators.py)
### Декоратор `log`
Этот декоратор предназначен для логирования информации о выполнении функций.
Он может записывать логи как в файл, так и выводить их в консоль.
### Основные функции
#### Логирование времени выполнения:

+ Записывает время начала и окончания выполнения функции.
+ Логирует результат выполнения функции.

#### Обработка ошибок:

+ При возникновении исключения записывает информацию об ошибке и входных параметрах функции.

#### Запись в файл или вывод на консоль:

+ Если предоставлен параметр filename, декоратор записывает логи в указанный файл.
+ Если параметр filename не задан, логи выводятся на консоль.

### Пример использования:
```python
from src.generators import log

@log(filename='logfile.txt')
def example_function(x, y):
    return x / y

example_function(10, 2)  # Запишет время выполнения и результат в файл 'logfile.txt'

@log()
def another_function(a, b):
    return a + b

another_function(5, 3)  # Выведет информацию о выполнении функции в консоль

```

## Модуль для чтения финансовых транзакций из файлов
### Описание
Этот модуль реализует функции для чтения и обработки финансовых транзакций из файлов форматов CSV и XLSX. 
Функции позволяют загружать данные из файлов и преобразовывать их в удобные для работы структуры данных (список словарей).

### Основные функции

1. `read_transactions_from_csv(file_path)`:

+ Читает данные из CSV-файла и возвращает их в виде списка словарей.
+ Используется библиотека pandas для эффективной работы с большими объемами данных.

2. `read_transactions_from_excel(file_path)`:

+ Читает данные из Excel-файла (формат XLSX) и возвращает их в виде списка словарей.
+ Поддерживается чтение файлов через движок openpyxl.

### Установка зависимостей
Для работы с файлами CSV и Excel необходимо установить следующие библиотеки:
```bash
pip install pandas openpyxl
```

### Пример использования
```python
from csv_excel import read_transactions_from_csv, read_transactions_from_excel

# Чтение данных из CSV
csv_transactions = read_transactions_from_csv('path/to/your/transactions.csv')

# Чтение данных из Excel
xlsx_transactions = read_transactions_from_excel('path/to/your/transactions.xlsx')

print(csv_transactions)
print(xlsx_transactions)
```

## Модуль `search_and_categorize.py` для поиска и категоризации банковских операций

### Описание
Этот модуль предоставляет функции для работы с данными о банковских операциях. 
Основные функции позволяют искать операции по описанию и категоризировать их по различным категориям. 
Модуль также поддерживает автоматическое тестирование с проверкой покрытия тестов.

### Основные функции
1. `search_operations(operations, search_string)`:

+ Ищет банковские операции, у которых в описании содержится указанная строка поиска.
+ Возвращает список операций, соответствующих условиям поиска.

2. `categorize_operations(operations, categories)`:

+ Категоризирует операции на основе описаний.
+ Возвращает словарь, где ключами являются категории, а значениями — количество операций в каждой категории.

### Пример использования
```python
from operations import search_operations, categorize_operations

# Данные о банковских операциях
operations = [
    {"description": "Перевод организации", "amount": 100},
    {"description": "Оплата коммунальных услуг", "amount": 50},
]

# Поиск операций с "организация"
search_result = search_operations(operations, "Перевод организации")

# Категоризация операций по описанию
categories = ["Перевод организации", "Оплата коммунальных услуг"]
category_result = categorize_operations(operations, categories)

print(search_result)
print(category_result)

```

## Модуль для поиска и категоризации банковских операций

### Описание

Этот модуль предоставляет функции для работы с данными о банковских операциях.
Основные функции модуля позволяют искать операции по описанию и категоризировать их по заранее заданным категориям.

### Основные функции
1. `search_operations(operations: list, search_string: str) -> list`:

+ Ищет банковские операции, где в описании встречается одно конкретное слово.
+ Поиск не зависит от регистра символов.
+ Возвращает список операций, где найдено слово в описании.

Пример использования:
```python
operations = [
    {"description": "Перевод организации", "amount": 100},
    {"description": "Оплата коммунальных услуг", "amount": 50},
    {"description": "Покупка продуктов", "amount": 200},
    {"description": "Перевод с карты на счет", "amount": 500},
]

result = search_operations(operations, "Перевод")
print(result)
```
Результат:
```python
[
    {"description": "Перевод организации", "amount": 100},
    {"description": "Перевод с карты на счет", "amount": 500}
]
```
2. `categorize_operations(operations: list, categories: list) -> dict`:

+ Категоризирует операции на основе ключевых слов в описании.
+ Возвращает словарь, где ключами являются категории, а значениями — количество операций, которые соответствуют каждой категории.
+ Поиск по категориям нечувствителен к регистру символов.

Пример использования:
```python
operations = [
    {"description": "Перевод организации", "amount": 100},
    {"description": "Оплата коммунальных услуг", "amount": 50},
    {"description": "Покупка продуктов", "amount": 200},
]

categories = ["организация", "коммунальные", "продукты"]
result = categorize_operations(operations, categories)
print(result)
```

Результат:
```python
{
    "организация": 1,
    "коммунальные": 1,
    "продукты": 1
}
```

## Логирование

### Описание

В проекте реализовано логирование для следующих модулей:
+ masks
+ utils

Логирование позволяет отслеживать важные события и ошибки, 
происходящие в процессе работы приложения. 
Логи включают временные метки, уровень серьезности и описание событий. 
Это помогает при отладке и анализе работы приложения.

### Формат логов
Логи записываются в текстовые файлы с расширением `.log`, которые находятся в папке `logs` в корне проекта. Каждый файл лога соответствует своему модулю:

+ Логи модуля `masks` записываются в файл `logs/masks.log.`
+ Логи модуля `utils` записываются в файл `logs/utils.log.`

Формат каждой записи в логах:
```php
<Время> - <Название модуля> - <Уровень серьезности>: <Сообщение>
```
### Пример записи лога
```yaml
2024-09-11 02:11:49,726 - utils - INFO: Применяется маска к номеру карты
```
### Уровни логирования
В проекте используются следующие уровни логирования:

+  INFO: Информирует о нормальном ходе работы программы, например, о применении маскировки к данным.
+ ERROR: Сообщает о возникших ошибках, например, если переданы некорректные данные (неверный номер карты или счета).

### Настройка логирования
Логирование настроено через модуль `logging`.
Для каждого модуля создаётся отдельный логер с настройкой на запись в соответствующий файл. 
Логи перезаписываются при каждом запуске приложения, что позволяет иметь актуальные записи событий.

Пример настройки логирования для модуля masks:
```python
import logging

logger = logging.getLogger("masks")
logger.setLevel(logging.INFO)
file_handler = logging.FileHandler("logs/masks.log", mode='w', encoding='utf-8')
file_formatter = logging.Formatter("%(asctime)s - %(name)s - %(levelname)s: %(message)s")
file_handler.setFormatter(file_formatter)
logger.addHandler(file_handler)
```

### Как использовать
Логеры автоматически записывают события во время работы приложения.
Вам не нужно предпринимать дополнительные шаги для их активации. 
Логи сохраняются в файлы при каждом запуске приложения, 
и их можно просмотреть в папке logs в корне проекта.

## Тестирование
Проект включает в себя набор тестов для проверки функциональности основных функций. Тесты написаны с использованием библиотеки pytest и включают как фикстуры, так и параметризацию.

#### Установка зависимостей для тестирования
Убедитесь, что у вас установлены все зависимости, необходимые для тестирования:
```bash
poetry install
```

#### Запуск тестов
Для запуска всех тестов выполните:
```bash
pytest
```

### Описание тестов
Тесты проверяют на коррекность функции представленные в проекте, включая нестандартные ситуации, по типу отсутвия некоторых данных или пустых входных данных.
